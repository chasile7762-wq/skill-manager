# Codex Skill 管理器用户使用手册

## 1. 产品简介

Codex Skill 管理器用于统一管理本机上的 Codex skill，覆盖三类来源：

- 系统预装 skill
- 用户全局 skill
- 项目级 skill

适合需要同时维护多个项目、多个 skill 来源的场景。

## 2. 主界面说明

### 2.1 顶部区域

- `安装 skill`：打开安装窗口，可从本地目录或 GitHub 安装
- `重新扫描`：重新扫描当前机器上的全部 skill

### 2.2 总览卡片

- `Skill 总数`：扫描到的全部 skill 数量
- `系统预装`：Codex 内置 skill 数量
- `全局`：用户级全局 skill 数量
- `项目级`：项目作用域 skill 数量
- `待处理问题`：缺少元数据、无效或重名冲突的 skill 数量
- `最近扫描`：最后一次扫描时间

### 2.3 Skill 列表

可按以下条件筛选：

- 搜索
- 来源
- 状态
- 项目

列表中可以直接：

- 查看 skill 详情
- 删除非只读 skill

### 2.4 项目面板

项目面板支持：

- 自动读取 Codex 已信任项目
- 手动补充项目路径
- 移除手动管理的项目

### 2.5 详情面板

详情面板会显示：

- 来源
- 状态
- 安装来源
- 是否只读
- 所属项目
- skill 路径
- `SKILL.md` frontmatter 预览

## 3. 安装 skill

### 3.1 从本地目录安装

1. 点击 `安装 skill`
2. 选择安装目标：`全局` 或 `项目`
3. 若选择项目，请先指定目标项目
4. 填写本地 skill 目录路径
5. 点击 `从本地目录安装`

### 3.2 从 GitHub 安装

1. 点击 `安装 skill`
2. 选择安装目标
3. 填写 GitHub tree 目录链接
4. 点击 `从 GitHub 安装`

支持的链接格式示例：

```text
https://github.com/owner/repo/tree/main/path/to/skill
```

## 4. 删除 skill

- 系统预装 skill 为只读，不能删除
- 全局 skill 和项目级 skill 可以删除

删除前建议确认该 skill 是否仍被工作流依赖。

## 5. 常见问题

### 5.1 为什么有些 skill 显示“重名冲突”？

表示当前机器上存在多个同名 skill，可能来自：

- 系统预装与全局同名
- 全局与项目级同名
- 多个项目内出现同名 skill

### 5.2 为什么有些 skill 显示“缺少元数据”？

通常说明 `SKILL.md` 中缺少：

- `name`
- `description`

### 5.3 为什么我看不到某个项目的 skill？

请确认：

- 项目路径已被添加
- 项目目录下存在 `.agents/skills`
- 点击过 `重新扫描`

## 6. 推荐使用顺序

1. 先扫描并查看总览
2. 添加需要管理的项目
3. 检查重名冲突与异常状态
4. 再安装新的 skill
5. 定期重新扫描
